Castor 1.3.3

Additions
- Moved documentation for the Castor JDO DDLGen to the reference guide, and deleted the corresponding HTML pages.
- Created a new menu section named 'Documentation' and moved all related menu items there.

Changes/Improvements
- Removed links to old releases from 'Main' menu section, and added links to old reference guide versions to the reference guide page.
- Reworked reference guide page to refer to old reference guide releases.

Castor 1.1.3

Additions
-Added information about how to configure Un-/Marshaller instances programmatically.
-Added documentation about the properties available for XML data binding.

Changes/Improvements
-Added mapping fragments/samples to XML mapping information.

Castor 1.0.3

Additions
-Added HOW-TO on mapping a java.util.List as a root element.

Changes/improvements
-Moved all documentation for the Castor XML code generator to its own main section named Castor CodeGen.
-Improved documentation for the Ant task definition for the Castor XML code generator.
-Moved documentation on code generator bindings into a separate page.
-Moved documentation on code generator properties into a separate page.

Castor 1.0.2 and earlier

Additions
-Added a page describing how to execute the new self-executable Castor JDO samples to the Castor JDO section. (Castor JDO)
-Added HOW-TO document on preparing a (unified) patch for submission. (Castor XML)
-Added HOW-TO document on mapping a collection of elements. (Castor XML)
-Added HOW-TO document on mapping a map/hashtable of elements. (Castor XML)
-Added a new page full of information relevant for people that want to contribute in one way or the other to the project.
-Added HOW-TO document on fetching XML dependencies from JAR files
-Added HOW-TO document on using identities and references in XML
-Added HOW-TO document on unmarshalling raw XML segments into arbitrary types

Changes/improvements
-Complete rewrite of information related to caching in Castor JDO. The new and improved documentation can be found here.
-Modified SVN/JIRA and various other areas of documentation to reflect migration of Castor repository from CVS to SVN.
-The download page now holds information about libraries required for using/building Castor.
-Modified XML Mapping documentation so that DOCTYPE public IDs match those defined in org.exolab.castor.util.DTDResolver
